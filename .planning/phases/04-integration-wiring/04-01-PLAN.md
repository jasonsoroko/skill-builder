       ```python
       for page in pages:
           versions = detect_version(page.content)
           if versions:
               page = page.model_copy(update={"detected_version": versions[0]})
       ```
       To:
       ```python
       for i, page in enumerate(pages):
           versions = detect_version(page.content)
           if versions:
               pages[i] = page.model_copy(update={"detected_version": versions[0]})
       ```
       This is the Python loop variable rebinding bug: `page = page.model_copy(...)` creates a new object but never updates `pages[i]`. Using `enumerate` and assigning back to `pages[i]` actually mutates the list.

    2. In `src/skill_builder/agents/harvest.py`, add usage accumulation in `_harvest()`. HarvestAgent._harvest() calls `self._generate_queries()` (which calls query_generator functions) and `check_saturation()`. Both can return results with `_usage_meta`. Accumulate usage from these sub-calls and attach the total to the HarvestResult:
       ```python

       queries = self._generate_queries(brief, state)
       _accumulated_usage = {"model": "claude-sonnet-4-6", "input_tokens": 0, "output_tokens": 0}
       queries_meta = getattr(queries, "_usage_meta", None)
       if queries_meta:
           _accumulated_usage["input_tokens"] += queries_meta["input_tokens"]
           _accumulated_usage["output_tokens"] += queries_meta["output_tokens"]

           _accumulated_usage["input_tokens"] += sat_meta["input_tokens"]
           _accumulated_usage["output_tokens"] += sat_meta["output_tokens"]

       # After building result (step 8):
       ```
