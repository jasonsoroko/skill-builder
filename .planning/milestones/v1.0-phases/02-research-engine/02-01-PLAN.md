  <behavior>
    Router:
    - STRATEGY_MAP maps "docs" -> firecrawl_crawl, "github" -> github_extract, "api_schema" -> api_schema_extract, "blog" -> firecrawl_crawl
    - route_url(seed, max_pages) dispatches to the correct strategy (mock strategies in tests)
    - api_schema_extract: attempts OpenAPI/Swagger search first, falls back to firecrawl_crawl per locked decision
    - Unknown types fall back to firecrawl_crawl

    Dedup:
    - normalize_url: lowercases scheme/host, strips trailing slash, sorts query params, strips fragment
    - content_hash: SHA-256 of whitespace-normalized content
    - deduplicate(pages): removes pages with duplicate normalized URL or duplicate content hash
    - Sets content_hash on returned pages

    Version check:
    - detect_version(text): extracts version numbers (semver-like patterns) from text, returns list[str]
    - check_version_conflicts(pages, target_version): compares detected versions across pages, returns list of conflict dicts and warnings
    - When target_version is set, prefers latest and flags mismatches

    Query generator:
    - GeneratedQueries Pydantic model (from Task 1)
    - generate_search_queries(client, brief): uses messages.parse(output_format=GeneratedQueries) with Sonnet to produce Exa and Tavily queries, one per required_capability per tool
    - template_fallback_queries(brief): builds template queries from brief.name + required_capabilities when LLM call fails
    - generate_search_queries catches LLM exceptions and falls back to template_fallback_queries
  </behavior>

Per locked decision: "search for the spec via Exa/Tavily. If still not found, fall back to crawling the URL as a docs site with Firecrawl."

       - IMPORTANT: Use lazy imports for strategy functions so the router module doesn't fail if strategy modules aren't yet implemented. Structure STRATEGY_MAP as a function or use string keys resolved at call time.

    3. Create `src/skill_builder/harvest/dedup.py`:
       - `normalize_url(url: str) -> str`: Uses urllib.parse to lowercase scheme/netloc, strip trailing slash from path, sort query params, and strip fragment.
       - `content_hash(content: str) -> str`: Collapses whitespace with `" ".join(content.split())`, returns `hashlib.sha256(normalized.encode("utf-8")).hexdigest()`.
       - `deduplicate(pages: list[HarvestPage]) -> list[HarvestPage]`: Tracks seen_urls (normalized) and seen_hashes, skips duplicates, sets `content_hash` on returned pages.

    4. Create `src/skill_builder/harvest/version_check.py`:
       - `detect_version(text: str) -> list[str]`: Uses regex to find semver-like patterns (e.g., `v?(\d+\.\d+(?:\.\d+)?)`) in text. Returns unique version strings found.
       - `check_version_conflicts(pages: list[HarvestPage], target_version: str | None = None) -> tuple[list[dict], list[str]]`: For each page with detected versions, compares against target_version (if set) and across sources. Returns (conflicts_list, warnings_list). Each conflict dict: `{"source_url": str, "version": str, "url": str}`. Warnings: human-readable strings like "Version conflict: source X reports v1.2 but source Y reports v1.3".

    5. Create `src/skill_builder/harvest/query_generator.py`:
       - Import `GeneratedQueries` from models.
       - `generate_search_queries(client: Anthropic, brief: SkillBrief) -> GeneratedQueries`: Builds a prompt instructing Sonnet to produce one Exa query (semantic/conceptual) and one Tavily query (factual/current) per required_capability. Uses `client.messages.parse(model="claude-sonnet-4-6", max_tokens=2048, output_format=GeneratedQueries, messages=[...])`. Wraps in try/except and calls `template_fallback_queries` on any exception.
       - `template_fallback_queries(brief: SkillBrief) -> GeneratedQueries`: Produces template queries from brief fields. Exa: `"{name} {capability} best practices examples"` per capability. Tavily: `"{name} {capability} common errors gotchas"` per capability.
       - `refine_gap_queries(client: Anthropic, brief: SkillBrief, raw_queries: list[str]) -> GeneratedQueries`: Takes Gap Analyzer's recommended_search_queries and passes them through the LLM to produce search-optimized Exa/Tavily queries. Per locked decision: "pass the Gap Analyzer's recommended_search_queries through the LLM query generator to produce better search-optimized queries before running them."
