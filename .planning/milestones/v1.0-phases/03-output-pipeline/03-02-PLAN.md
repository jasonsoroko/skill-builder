<objective>
Build the LLM-as-judge evaluators (API accuracy, completeness, trigger quality), the ValidatorAgent that orchestrates all 5 evaluators with fail-fast and parallel execution, and wire everything into the conductor with proper feedback routing.

Purpose: This is the quality gate. Heuristic evaluators (from Plan 01) catch obvious issues cheaply. LLM evaluators use Opus to verify factual accuracy, completeness, and trigger quality. The ValidatorAgent fail-fast pattern saves cost by skipping Opus calls when heuristics fail. The conductor wiring ensures failed dimensions route back to the Mapper for re-production.
Output: 3 LLM evaluators, ValidatorAgent, conductor wired with real validator and feedback kwargs.
</objective>
